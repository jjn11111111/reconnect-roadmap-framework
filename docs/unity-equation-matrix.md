## Crossover — when product overtakes sum

For repeated **same factor** (e.g. **1.111…** ≈ 10/9, or any base **> 1**):

| n repeats | Sum (additive) | Product (multiplicative) |
|-----------|----------------|--------------------------|
| Low n | Sum **grows linearly**: n × a | Product **starts near** a |
| High n | Still linear | **Exponential** — overtakes sum |

**Example sketch** (a = 1.111…):

```
n = 2:   sum ≈ 2.22    product ≈ 1.23   → sum larger
n = 10:  sum ≈ 11.1    product ≈ 2.59   → sum larger
n = 100: sum ≈ 111     product ≈ 13,780 → product vastly larger
```

**Framework read:**

- **Early generations:** **Addition seems to "win"** — 1+1=2 vs 1×1=1 — matrix uses this to **privilege** pole-splitting, counting, competition.
- **Late generations:** **Multiplication overtakes** in magnitude — doubling path, compound interest, viral scale — matrix uses this to **privilege** empire, extraction, "winner takes all."

**Middle node:** Neither operation is **evil**; **either alone** is half-waveform. **Fractal circuit** uses **both** — Fibonacci **and** doubling — with **`1×1=1×1`** as **origin seal**.
